从零开始：用GitHub Pages搭建个人博客

发表于2026-04-21|更新于2026-04-21|教程
|总字数:1.7k|阅读时长:6分钟|浏览量:

前言

拥有一个个人博客，是记录技术成长、分享经验心得的最佳方式之一。而 GitHub Pages 提供了免费、稳定的静态网站托管服务，配合 Hexo 这样的静态博客框架，你可以零成本搭建一个属于自己的技术博客。

本教程将从零开始，手把手带你完成整个搭建过程。

你将获得：

  • 一个免费的个人博客网站（yourname.github.io）
  • 支持 Markdown 写作
  • 美观的 Butterfly 主题
  • 一键部署到 GitHub Pages

常见问题

Q: 部署后页面样式丢失？

确认 _config.yml 中的 url 配置正确，执行 hexo clean 后重新生成。

Q: 如何绑定自定义域名？

在 GitHub 仓库的 Settings → Pages 中添加自定义域名，并在 source/ 目录下创建 CNAME 文件，写入你的域名。

Q: 文章如何设置封面图？

在文章的 front-matter 中添加 cover: 图片路径。

Q: 如何更新主题？

由于你使用的是 _config.butterfly.yml 覆盖配置，更新不会影响你的自定义设置。

总结

恭喜你完成了个人博客的搭建！整个流程回顾：

  1. 安装 Node.js、Git、Hexo
  2. hexo init 初始化项目
  3. 创建 GitHub 仓库并配置部署
  4. 安装 Butterfly 主题
  5. 配置站点信息和页面
  6. 开始写作并发布

写作是最好的学习方式。 不要追求完美，先写起来，在写作中不断成长。
